# 关卡怪物系统解耦迭代

## Goal

将关卡与怪物系统解耦，建立独立的怪物定义体系（属性/技能/等级可成长），重构关卡为多区域多关卡结构，扩展战斗系统支持多怪物与目标选择，并优化整体数值平衡（含等级压制系统）。

## Confirmed Facts (from codebase)

### 当前关卡系统
- `packages/shared/src/constants/levels.ts`：3个关卡（史莱姆平原/骸骨墓地/恶魔巢穴）
- 每关4波（3小怪+1Boss），每波 `monsters: MonsterDef[]` 数组但始终只有1个元素
- 怪物定义内联在关卡中，无独立怪物模板
- `LevelDef` 类型：id, name, desc, difficulty, recommendLevel, waves, firstClearReward

### 当前怪物类型（两套不一致系统）
- `MonsterDef`（types/index.ts）：内联在关卡波次中，含 baseStats/level/tags/icon/isBoss/loot
- `Monster`（types/index.ts）+ `server/services/templates.ts`：独立模板系统，含 hp/maxHp/stats/lootTable，有 `getMonsterForStage()` 但未接入实际战斗
- 战斗实际使用的是 `MonsterDef`（关卡内联），`templates.ts` 疑似遗留系统

### 当前战斗系统
- `TurnState` 严格1v1：单个 enemyHp/enemyMaxHp/enemyDef/enemyAtk/enemySpeed/enemyDerived
- `initTurnState()` 只取 `wave.monsters[0]`
- 波次推进：enemyHp <= 0 时 waveIndex++，客户端调 `advanceWave()` 初始化新 TurnState
- `multi_target` 技能效果只是给同一敌人加额外伤害，非真正多目标
- 怪物无技能系统，Boss 只是 `enemyIsBoss` 标记 + 随机倍率攻击

### 当前数值系统
- 等级加成：`1 + (level - 1) * 0.05`（每级+5%攻防血）
- 减伤公式：`defense / (defense + K)`，K=100
- 无等级压制系统
- `calcDerivedStats(baseStats, equipment, level)` 统一计算英雄和怪物二级属性

### 当前冒险UI
- `AdventureMap.tsx`：硬编码3个节点坐标，SVG蜿蜒路径
- 无区域/大地区概念
- 关卡进度 `LevelProgress`：cleared/stars/firstClearClaimed

### 存档系统
- `gameStore.ts` 有 stage/highestStage 字段（疑似遗留），levelProgress 按 levelId 索引
- localStorage 存档，save/load 逻辑在 gameStore

## Requirements

### R1: 怪物系统解耦
- 怪物独立定义，拥有自己的属性、技能和等级
- **所有怪物都有主动技能**（非仅Boss）
- **怪物技能无资源限制**：每回合自由选择技能，AI决定最优策略（无怒气/冷却限制）
- **怪物AI采用效用评分制**：AI对每个可用技能按当前局势评分（伤害潜力、生存、控制等），选最高分技能。加入随机性避免完全可预测
- **属性逐属性线性成长**：每个怪物模板定义各属性成长率（如力+1.5/级），等级N时 = baseStat + growth*(N-1)
- 怪物技能跟随等级成长（具体机制待定）
- 关卡只需引用怪物模板ID + 设置等级即可

### R2: 多区域关卡结构
- **3个大地区**，每个含10个关卡（共30关）
- 区域可复用现有主题：史莱姆平原(1-x) / 散骨墓地(2-x) / 恶魔巢穴(3-x)
- 关卡显示格式：1-1, 1-2, ... 1-10, 2-1, ...
- 第5关和第10关可设为Boss关

### R3: 数值平衡与等级压制
- 角色+装备属性对应相对等级有合理战斗体验
- **等级压制采用伤害倍率机制**：在伤害计算环节加入等级差修正倍率（高等级打低等级加伤，低等级打高等级减伤）
- 属性系统保持现有成长曲线，等级压制独立生效
- 具体倍率参数待定（预计每级 ±5~8%）

### R4: 多怪物战斗与目标选择
- 关卡波次允许多个怪物同时出场
- **波次结构灵活设计**：每关独立配置波次组成（如1~4关=2波小怪+1波精英，第5关=Boss单独战，后续循环类似模式）
- 不要求一次打完10关，每关独立进入
- **战斗单位**：英雄单人对战N怪物（本次不加入友军参战，数据结构预留友军扩展）
- **回合顺序**：英雄行动→所有存活敌人依次行动（经典单英雄回合制）
- **目标选择流程**：点技能按钮→进入选目标模式→点敌人确认释放。提供“自动选择目标”开关供快速操作
- 技能目标模式：手动选择（如治疗N个友军）vs 自动连锁（如飞盾连锁，不可选）
- 需平衡英雄面对多敌时的生存能力

### R5: 英雄技能系统适配
- 重构现有10个战士技能适配多目标系统（如旋风斩→真正 AoE/连锁）
- 新增 2-3 个展示新机制的技能（如飞盾=连锁、AoE技能）
- 技能新增目标模式字段：单体/连锁/多选/AOE/自身

## Open Questions

- 区域解锁机制：假设顺序解锁（清完1区域解锁2区域，区域内顺序解锁关卡）
- 怪物模板数量：预计 12-15 个（每区域 4-5 个，部分跨区域复用）
- 等级压制具体倍率：预计每级 ±7%，上限 ±60%
- 怪物技能数量：普通怪 2-3 个，精英 3-4 个，Boss 4-5 个
- 以上具体数值在 design.md 中提出，可调优

## Acceptance Criteria

- [ ] 怪物有独立定义文件，含属性/技能/等级成长逻辑（逐属性线性成长）
- [ ] 所有怪物都有主动技能，AI 采用效用评分制选择技能
- [ ] 关卡引用怪物模板ID+等级，不再内联完整怪物数据
- [ ] 存在3个大地区，每个含10个关卡，显示为 1-1 ~ 1-10 格式
- [ ] 关卡波次结构灵活设计（如1~4关=2波小怪+1波精英，5关=Boss）
- [ ] 战斗系统支持同一波次多个怪物同时存在
- [ ] 回合顺序：英雄行动→所有存活敌人依次行动
- [ ] 玩家可选择攻击目标（点技能→选目标→释放），支持自动选择开关
- [ ] 技能目标模式：单体/连锁/多选/AOE/自身
- [ ] 现有技能重构适配多目标系统，新增2-3个展示新机制的技能
- [ ] 等级压制系统生效（伤害倍率机制）
- [ ] 存档重置（项目开发阶段，无旧存档需迁移）
- [ ] 清理遗留的 server/templates.ts 和 Monster 类型

## Out of Scope

- 友军/宠物参战（数据结构预留接口）
- 新职业（仅适配现有战士）
- PVP/排行榜
- 关卡内事件/商店/休息点等Roguelike元素
- 自动战斗/扫荡功能
